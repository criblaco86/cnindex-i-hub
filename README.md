# cnindex-i-hub

本仓库用于归档和发布多个独立的 HTML 页面。这些页面由社区成员或维护者制作，内容涵盖各类工具、信息聚合与静态展示页面。

## 项目简介

cnindex-i-hub 是一个静态页面集合仓库。不同于单一网站项目，这里汇集了多个彼此独立、功能各异的 HTML 文件。每个页面都有其自身的用途与设计，适合直接通过 GitHub Pages 或其他静态托管服务部署。

## 目录说明

```
.
├── index.html          # 仓库主页（可自定义）
├── pages/              # 存放各独立 HTML 页面
│   ├── page1.html
│   ├── page2.html
│   └── ...
├── assets/             # 公共资源（CSS、JS、图片等）
│   ├── css/
│   ├── js/
│   └── images/
└── README.md           # 本文件
```

- `pages/`：每个 HTML 文件为一个独立页面，命名应具有辨识度。
- `assets/`：存放页面共用的样式、脚本或图片资源，避免重复。
- 如有需要，可自行在根目录添加其他文件夹或文件。

## 页面归档说明

- 所有页面均以静态 HTML 形式存在，不依赖后端服务。
- 页面之间无强制关联，可单独访问或部署。
- 提交新页面时，请确保文件结构清晰，并更新 `pages/` 目录下索引（如有）。
- 不建议在页面中嵌入跟踪代码、外部请求或动态内容，以保持纯粹静态。

## 维护说明

本仓库由维护者定期审核并合并提交。欢迎通过 Issues 或 Pull Requests 提交新页面或改进建议。

- **提交新页面**：将 HTML 文件放入 `pages/` 目录，并在 PR 描述中简要说明页面用途。
- **修复或更新**：直接修改对应文件，并注明改动原因。
- **资源管理**：若页面使用独立资源，请放在 `assets/` 对应子目录下，避免散落。

## 部署

本仓库默认启用 GitHub Pages，发布内容为根目录文件。如需自定义部署方式，请修改仓库设置或添加 `.github/workflows/` 配置文件。
